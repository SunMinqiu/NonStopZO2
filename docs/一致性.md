**结论**
默认主路径里，最关键的“锁 / 阻塞 / 一致性”点是这 6 个：

1. 训练启动默认等 `shadow_ready_event`，会 block startup。  
2. 每个训练 step 默认经过 `update_lock`，是热路径串行点。  
3. 优化器默认有多处 `torch.cuda.synchronize()` / stream synchronize，是训练吞吐的硬阻塞源。  
4. shadow 默认 `commit_interval=1`，每步 commit，shadow 吞吐默认就被 durable commit 限制。  
5. shadow snapshot 有完整的 `writing -> ready + sha256` 协议，读取时也严格校验；这是默认一致性核心。  
6. checkpoint / recover / resume 都会做大对象 clone、replay、恢复校验；不在每步，但一旦触发就是重阻塞。
